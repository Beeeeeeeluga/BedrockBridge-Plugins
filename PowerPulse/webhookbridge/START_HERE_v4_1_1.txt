================================================================================
ISSUES THAT ARE NOW FIXED
================================================================================

ISSUE #1: "Entity death tracking error"
STATUS: ✅ FIXED
What it was: When entities died, sometimes the plugin crashed trying to
            access the location. This happened because Bedrock removes
            entities asynchronously.
What changed: Now safely handles missing entity locations with fallback {0,0,0}
Result: Events logged reliably, no more crashes

ISSUE #2: "Rate limited (429), retrying..."
STATUS: ✅ FIXED
What it was: When Discord said "you're sending too many messages", the
            plugin didn't respect the timing it suggested. This caused
            more failed retries.
What changed: Now reads Discord's "Retry-After" header and waits the proper
            amount of time. Uses exponential backoff (1s, 2s, 4s, etc.)
Result: Better success rate, fewer repeated failures

CONSOLE SPAM:
STATUS: ✅ FIXED
What it was: Entity errors were logged as ERROR level, causing warning spam
What changed: Changed to WARN level (expected behavior)
Result: Cleaner, less spammy logs
