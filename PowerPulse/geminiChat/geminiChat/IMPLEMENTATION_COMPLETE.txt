HOW IT WORKS NOW
================================================================================

Step-by-Step Flow:

1. Player asks: @g What is Minecraft?

2. Plugin processes:
   - Validates input
   - Gets conversation history
   - Sends to Gemini API
   - Receives response

3. Bedrock sees:
   [Gemini] Here's what Minecraft is...

4. Discord receives:
   Beautiful embed with:
   - Title: Gemini AI Response
   - Question: What is Minecraft?
   - Answer: Here's what Minecraft is...
   - Player: [name]
   - Server: Minecraft Bedrock
   - Color: Purple
   - Footer: Gemini AI Chat Plugin

5. Console logs:
   [DISCORD] Sending Gemini response to Discord from [Player]
   [Embeds] Creating Gemini response embed for [Player]
   [DiscordBridge] BridgeDirect ready, sending embed directly
   [DiscordBridge] Response embed sent to Discord successfully from [Player]

================================================================================
API RESPONSE DEBUGGING
================================================================================

The httpClient.js has enhanced debugging for API responses:

When API returns status 200 but response seems empty:
- Console shows: "[Gemini Response Debug] Full Response: [JSON]"
- Tries multiple parsing paths:
  1. candidates[0].content.parts[0].text (primary)
  2. result.candidates[0].content.parts[0].text (alternative)
  3. Direct string (fallback)
- If still empty: Shows full JSON for investigation
- This helps identify exact response structure
